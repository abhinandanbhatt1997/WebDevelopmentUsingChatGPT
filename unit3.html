Unit 3: Graph Algorithms

Overview of Topics

This unit covers key graph algorithms, such as Dijkstra’s Algorithm, DFS, and BFS. Understanding these algorithms will enable you to solve real-world network problems.

BFS explores nodes layer by layer, making it ideal for finding the shortest path in unweighted graphs. BFS is commonly used for finding connected components and shortest paths.

Example Code:

from collections import deque

def bfs(graph, start):
    queue = deque([start])
    visited = set()
    while queue:
        node = queue.popleft()
        if node not in visited:
            visited.add(node)
            queue.extend(graph[node] - visited)

DFS explores as far as possible along a branch before backtracking. It is useful for detecting cycles and finding connected components.

Example Code:

def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    for next in graph[start] - visited:
        dfs(graph, next, visited)
    return visited

Dijkstra's Algorithm finds the shortest path from a starting node to all other nodes in a graph with non-negative weights, commonly used in GPS and network routing.

Example Code:

import heapq

def dijkstra(graph, start):
    distances = {vertex: float('infinity') for vertex in graph}
    distances[start] = 0
    pq = [(0, start)]
    while pq:
        current_distance, current_vertex = heapq.heappop(pq)
        if current_distance > distances[current_vertex]:
            continue
        for neighbor, weight in graph[current_vertex].items():
            distance = current_distance + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))
    return distances

Interview Questions

Q1: What is the main difference between BFS and DFS?

Answer: BFS explores nodes layer by layer and is optimal for finding the shortest path in unweighted graphs. DFS, however, explores as far as possible down a branch and is useful for detecting cycles and finding connected components.

Q2: When is Dijkstra’s Algorithm unsuitable for shortest-path problems?

Answer: Dijkstra’s Algorithm is unsuitable when the graph has negative weight edges, as it may not produce correct results.

Q3: What are practical applications of graph algorithms in the real world?

Answer: Graph algorithms are used in applications like GPS navigation (shortest path), social networks (suggesting connections), web crawling (exploring linked pages), and computer networking (routing).
