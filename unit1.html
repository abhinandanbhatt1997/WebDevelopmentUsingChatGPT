Arrays are a basic data structure where elements are stored in contiguous memory. They allow efficient access by index but are fixed in size.

Key Characteristics:

  • Fixed Size: Once declared, the size cannot change.
  • Indexed Access: Elements can be accessed in constant time O(1).
  • Homogeneous: All elements must be of the same data type.

Example:

In Python: arr = [1, 2, 3, 4, 5]

Linked Lists are a collection of nodes where each node points to the next, making them dynamic in size and useful for certain types of insertion and deletion operations.

Key Characteristics:

  • Dynamic Size: Can grow and shrink as needed.
  • Non-contiguous Memory: Nodes can be stored anywhere in memory.
  • Nodes: Each node contains data and a pointer to the next node.

Example:

In Python:

class Node:
  def __init__(self, data):
    self.data = data
    self.next = None

Stacks follow a Last In, First Out (LIFO) structure. They are used in recursive algorithms and undo functionalities.

Key Characteristics:

  • LIFO Order: The last element added is the first one removed.
  • Push and Pop: Operations to add and remove elements.
  • Stack Overflow: Occurs when trying to add an element to a full stack.

Example:

In Python:

stack = []
stack.append(1) # Push
stack.pop() # Pop

Queues follow a First In, First Out (FIFO) structure and are commonly used in scheduling tasks and managing data buffers.

Key Characteristics:

  • FIFO Order: The first element added is the first one removed.
  • Enqueue and Dequeue: Operations to add and remove elements.
  • Circular Queue: A circular structure that improves space efficiency.

Example:

In Python:

from collections import deque
queue = deque()
queue.append(1) # Enqueue
queue.popleft() # Dequeue

Time Complexity is the measure of the time an algorithm takes to complete as a function of the input size. It is often expressed using Big O notation, like O(n), O(log n).

Common Complexities:

  • O(1): Constant time
  • O(log n): Logarithmic time
  • O(n): Linear time
  • O(n log n): Linearithmic time
  • O(n^2): Quadratic time

Example:

A linear search has a time complexity of O(n).

Space Complexity measures the amount of memory an algorithm requires relative to the input size.

Key Factors:

  • Input Space: Memory required to store input data.
  • Auxiliary Space: Extra space used by the algorithm, excluding input space.
  • Recursive Calls: Stack space used during recursion.

Example:

A recursive function with O(n) space complexity due to the call stack.

Interview Questions

Arrays

  • What are the differences between arrays and linked lists?
    **Answer:** Arrays are fixed in size and allow random access to elements, while linked lists are dynamic and consist of nodes that point to the next element, allowing efficient insertion and deletion.
  • How would you find the largest element in an array?
    **Answer:** Iterate through the array, comparing each element to find the largest.

Linked Lists

  • How do you reverse a linked list?
    **Answer:** Iterate through the list and adjust the pointers to reverse the direction of the links.
  • What are the advantages of linked lists over arrays?
    **Answer:** Linked lists allow for dynamic sizing and efficient insertions and deletions at any position.

Stacks

  • Explain the use of stacks in programming.
    **Answer:** Stacks are used for function calls, backtracking algorithms, and undo mechanisms.
  • How can you implement a stack using an array?
    **Answer:** Use an array to store the elements and maintain an index to represent the top of the stack.

Queues

  • What is a circular queue?
    **Answer:** A circular queue connects the end of the queue back to the beginning, optimizing space utilization.
  • How can you implement a queue using two stacks?
    **Answer:** Use two stacks to manage enqueue and dequeue operations, transferring elements as needed.
